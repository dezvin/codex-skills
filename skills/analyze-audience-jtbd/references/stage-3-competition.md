# Этап 3. Конкурентный контекст

## Цель

Понять, какими способами клиент уже выполняет работу, почему сохраняет
статус-кво и как triggers влияют на выбор категории решения.

## Вход

Использовать утвержденный consolidated set JTBD, карту `сегмент -> работа`,
search triggers и доступные факты о текущем поведении клиентов.

## Порядок

Для общего поля работ и существенных различий между ними определить:

1. **Прямые конкуренты:** решения той же категории, выполняющие ту же работу
   схожим способом.
2. **Вторичные и альтернативные решения:** способы, закрывающие часть работы
   или выполняющие ее другим механизмом.
3. **DIY:** ручные, самодельные и неформальные способы клиента.
4. **Не-потребление / статус-кво:** ничего не делать, терпеть проблему или
   продолжать неудобный текущий способ.

Для каждого способа показать:

- какую часть работы он выполняет;
- почему клиент его выбирает;
- что в нем устраивает и не устраивает;
- какие усилия и риски связаны с переключением;
- какие triggers усиливают или ослабляют вероятность выбора.

Проверить переходы:

- статус-кво -> DIY;
- статус-кво или DIY -> специализированное решение;
- один тип решения -> другой;
- новое решение -> возврат к привычному способу.

## Результат

Структурированная карта четырех типов конкурирующих решений с привязкой к
jobs, сегментам и triggers. Явно отделить известное поведение клиентов от
предполагаемых альтернатив.

## Проверка

- Конкуренты определены через работу клиента, а не только через товарную
  категорию.
- Статус-кво и DIY не потеряны.
- Карта достаточно конкретна для анализа критериев выбора и возражений.
- Рыночные утверждения не выданы за факты без источника.

Запросить подтверждение карты перед глубокой детализацией Этапа 4.
