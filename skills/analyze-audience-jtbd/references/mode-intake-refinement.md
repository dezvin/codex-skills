# Уточнение входа

## Когда использовать

Использовать до выбора аналитического режима, если запрос неясен, смешивает
несколько задач или не позволяет понять объект и цель анализа.

Не использовать как обязательную анкету. Если маршрут понятен, сразу перейти
к нему без отдельного блока уточнения.

## Задача

Определить:

1. Что анализируется: продукт, аудитория, клиентский сигнал, набор материалов,
   существующий JTBD-файл или отдельный вывод.
2. Какое решение пользователь хочет принять: понять аудиторию, найти jobs,
   проверить сегмент, разобрать возражение, изменить предложение, обновить
   JTBD, получить рекомендации или провести полный анализ.
3. Какие материалы уже есть.
4. Какой минимальный режим даст полезный ответ.

## Порядок

1. Сохранить исходную формулировку пользователя без исправления смысла.
2. Классифицировать вход:
   - один клиентский сигнал;
   - продукт с неизвестной аудиторией;
   - один или несколько предварительных сегментов;
   - набор материалов;
   - существующий JTBD и новые данные;
   - продолжение анализа;
   - локальный вопрос к отдельному этапу.
3. Проверить минимальную достаточность для выбранного маршрута.
4. Задать только вопросы, ответы на которые изменят маршрут или существенно
   изменят анализ.
5. Зафиксировать явные допущения по несущественным пробелам.
6. Назвать выбранный режим и начать его либо дождаться критически необходимого
   ответа.

## Минимальные вопросы

Выбирать только нужные:

- Что именно вы хотите понять или решить по итогам?
- Есть ли существующий JTBD-файл, с которым нужно сравнить наблюдение?
- Это единичный случай или похожее уже повторялось?
- Какой продукт, услуга или рабочая ситуация рассматривается?
- Какие материалы нужно считать исходными данными?

Не спрашивать о размере рынка, демографии, персоне, бюджете и готовности
платить «на всякий случай».

## Маршрутизация

- Один сигнал без задачи изменить файл -> `mode-client-signal.md`.
- Один или несколько сигналов плюс существующий JTBD ->
  `mode-client-signal.md` и `mode-existing-jtbd-refinement.md`.
- Набор интервью или заметок -> `mode-material-analysis.md`.
- Создание анализа с нуля -> `stage-1-framing.md`.
- Готовая рамка и сырой материал -> `stage-2-jobs.md`.
- Локальный вопрос -> reference соответствующего этапа.
- Продолжение -> `state-and-resume.md` и reference текущего этапа.

## Результат

Кратко зафиксировать только полезное:

```text
Объект анализа
Цель анализа
Что уже известно
Какие материалы есть
Что пока предполагается
Чего критически не хватает
Выбранный режим
С какого этапа начинаем
```

Для полного анализа эти сведения могут стать началом рабочего файла после
подтверждения пользователя. Для короткого сигнала не создавать отдельный файл.

## Границы

- Не проводить глубокий анализ внутри уточнения входа.
- Не заставлять пользователя заполнять длинную форму.
- Не требовать готовую аудиторию заранее.
- Не задавать вопросы, не влияющие на ближайший шаг.
- Не записывать долговечные файлы на этом режиме.
