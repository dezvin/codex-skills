# Codex Skills

Рабочие skills для Codex, которые помогают не терять контекст, не решать
неверно поставленную задачу, сохранять проектную работу и проектировать
изменения до реализации.

Каждый skill устанавливается отдельно. Можно собрать только тот набор, который
нужен для вашей работы.

## Выберите skill по ситуации

| Если Codex… | Используйте |
| --- | --- |
| Уверенно решает не ту задачу или застрял в слишком узкой рамке | [`zoom-out`](skills/zoom-out) |
| Должен перенести выбранную задачу или проект в новый чат без потери состояния | [`handoff`](skills/handoff) |
| Накопил длинное обсуждение, которое надо превратить в проект реализации | [`design-pass`](skills/design-pass) |
| Должен выявить значимые развилки в идее, плане или решении | [`grill-me`](skills/grill-me) |
| Должен сохранить незавершённую проектную работу между сессиями | [`manage-project-tasks`](skills/manage-project-tasks) |
| Не может честно определить весь маршрут заранее | [`nearest-clarity`](skills/nearest-clarity) |
| Должен объединить или сократить разросшиеся документы | [`document-consolidator`](skills/document-consolidator) |
| Должен явно сохранить наблюдаемую историю чата, вызовы и результаты инструментов в `.txt` | [`export-current-thread`](skills/export-current-thread) |
| Должен исследовать аудиторию через Jobs to be Done | [`analyze-audience-jtbd`](skills/analyze-audience-jtbd) |
| Должен подготовить переносимый промпт для генерации изображения | [`universal-visual-prompt-builder`](skills/universal-visual-prompt-builder) |
| Должен безопасно изменить, опубликовать или восстановить сайт на Beget | [`manage-beget-site`](skills/manage-beget-site) |

## Быстрая установка

Попросите Codex:

```text
Use $skill-installer to install a skill from:
https://github.com/dezvin/codex-skills/tree/main/skills/<skill-name>
```

Например:

```text
Use $skill-installer to install a skill from:
https://github.com/dezvin/codex-skills/tree/main/skills/handoff
```

Или установите из PowerShell:

```powershell
python "$env:USERPROFILE\.codex\skills\.system\skill-installer\scripts\install-skill-from-github.py" `
  --repo dezvin/codex-skills `
  --path skills/handoff
```

После первой установки перезапустите Codex, чтобы skill появился в списке
доступных.

## Каталог

| Skill | Что делает | Где устанавливать |
| --- | --- | --- |
| [`zoom-out`](skills/zoom-out) | Проверяет рамку, цель, факты, допущения и риски до продолжения работы. | Глобально |
| [`handoff`](skills/handoff) | Переносит выбранную работу в новый чат: создаёт компактный handoff-файл или самодостаточный промпт. | Глобально |
| [`design-pass`](skills/design-pass) | Разбирает принятые решения и готовит проект реализации без прямого внедрения. | Глобально |
| [`grill-me`](skills/grill-me) | Выявляет пользовательские развилки одним важным вопросом за раз и не перекладывает на пользователя факты, обратимые мелочи и проверяемые гипотезы. | Глобально |
| [`manage-project-tasks`](skills/manage-project-tasks) | Ведёт долговременную проектную работу в `TODO.md` и месячном архиве. | Глобально или в проекте |
| [`nearest-clarity`](skills/nearest-clarity) | Определяет ближайший завершённый участок при настоящей неопределённости и показывает дальнейший горизонт без ложной точности. | Глобально или в проекте |
| [`document-consolidator`](skills/document-consolidator) | Безопасно сжимает и объединяет `.md` и `.txt`, сохраняя факты, решения, ссылки и незавершённую работу. | Глобально |
| [`export-current-thread`](skills/export-current-thread) | По явной команде сохраняет хронологию сообщений, вызовов и полных текстовых результатов инструментов в локальный `.txt`. | Глобально |
| [`analyze-audience-jtbd`](skills/analyze-audience-jtbd) | Проводит полный или частичный JTBD-анализ по материалам, интервью и клиентским сигналам. | Глобально |
| [`universal-visual-prompt-builder`](skills/universal-visual-prompt-builder) | Сам выбирает подходящее художественное направление и превращает его в переносимый визуальный промпт с убедительной техникой исполнения. Изображения не генерирует. | Глобально |
| [`manage-beget-site`](skills/manage-beget-site) | Исследует реальный live-контур, готовит бэкап и откат, изменяет сайт и проверяет результат на домене. | Глобально |

Глобальный skill доступен в любых проектах. Проектный skill лежит в
`.agents/skills/` конкретного проекта и действует только там. Место установки
`manage-project-tasks` и `nearest-clarity` выбирает пользователь; изменения
`AGENTS.md` для них не нужны.

`export-current-thread` — явная утилита. Она не должна включаться по мягким
формулировкам и молча создавать экспорт разговора.

## Проектная установка

Чтобы установить skill только в текущий проект, укажите `.agents/skills` как
место назначения:

```powershell
python "$env:USERPROFILE\.codex\skills\.system\skill-installer\scripts\install-skill-from-github.py" `
  --repo dezvin/codex-skills `
  --path skills/manage-project-tasks `
  --dest .agents\skills
```

Для `nearest-clarity` меняется только путь:

```powershell
python "$env:USERPROFILE\.codex\skills\.system\skill-installer\scripts\install-skill-from-github.py" `
  --repo dezvin/codex-skills `
  --path skills/nearest-clarity `
  --dest .agents\skills
```

Дополнительный установщик, конфигурационный файл или раздел в `AGENTS.md` не
требуется.

## Как обновлять установленные skills

Папка в этом репозитории — опубликованный источник. Установленный skill в
`.codex/skills` или `.agents/skills` — отдельная копия: после обновления
репозитория она не обновляется автоматически.

Стандартный установщик не перезаписывает существующую папку. Для безопасного
обновления попросите Codex сначала сравнить опубликованную и установленную
версии, а затем заменить только файлы выбранного skill:

```text
Обнови установленный <skill-name> из:
https://github.com/dezvin/codex-skills/tree/main/skills/<skill-name>

Сначала сравни версии. Обнови только папку этого skill, не трогай другие
skills и проектные файлы. После обновления проверь структуру и сообщи, какие
файлы изменились.
```

Если старый установщик когда-то добавил инструкции в проектный `AGENTS.md`,
они не исчезнут автоматически. Их нужно проверить и удалить отдельно, не
затрагивая остальные проектные правила.

## Что важно знать о сложных skills

### `handoff`

`handoff` сначала определяет, какую задачу, проект или часть работы нужно
перенести. В Codex он может временно восстановить выпавшую из видимого
контекста историю текущего чата, но в итоговый handoff включает только
выбранный рабочий контур. Сырой экспорт удаляется и не передаётся дальше.

При доступной файловой системе итог сохраняется в `.handoffs/` проекта. Если
создать файл нельзя, skill возвращает самодостаточный промпт. Следующий чат
получает текущую задачу и состояние, но не наследует лишние разрешения на
внешние, разрушительные или расширяющие область действия операции.

### `design-pass`

`design-pass` сначала устанавливает, что пользователь действительно принял,
отверг, заменил или оставил открытым. Затем он либо останавливается на сверке
решений, либо превращает подтверждённые решения в требования, ограничения,
карту влияния, проверки и готовую команду для реализации в новом чате.

Он не принимает предложения ассистента за решения пользователя, не создаёт
экспорт разговора без согласия и не реализует спроектированные изменения сам.

### `manage-project-tasks`

`manage-project-tasks` сохраняет только ту проектную работу, которая должна
пережить текущую сессию. Краткосрочные действия, обсуждения и уже завершённая
работа в `TODO.md` не попадают.

При первой реальной записи skill сам создаёт корневой `TODO.md`, если
совместимой системы задач ещё нет. Закрытые задачи сохраняются в ленивом
архиве `archive/tasks/YYYY-MM.md`; пустой архив заранее не создаётся.

### `nearest-clarity`

`nearest-clarity` нужен не для любой сложной работы, а для настоящей
неопределённости: когда важные сведения появятся только в ходе исполнения,
остаются жизнеспособные направления или новые данные могут изменить основание
работы.

Он показывает целевой контур, ближайший ограниченный участок, результат его
завершения, первое действие, предварительно следующие участки и дальний
горизонт. Дальняя часть пути остаётся предположительной, пока для неё не
появятся основания.

## Происхождение и адаптации

`handoff` основан на минимальном productivity skill Мэтта Покока:
[`handoff`](https://github.com/mattpocock/skills/blob/main/skills/productivity/handoff/SKILL.md).
Эта версия дополнительно выбирает переносимый рабочий контур, восстанавливает
доступную историю Codex при необходимости, фиксирует границу полномочий и
готовит команду для нового чата. Она также поддерживает неинженерные задачи.

`zoom-out` вдохновлён инженерным skill Мэтта Покока:
[`zoom-out`](https://github.com/mattpocock/skills/blob/main/skills/engineering/zoom-out/SKILL.md).
Эта версия работает не только с кодом, но и со стратегией, исследованиями,
маркетингом, контентом, документами и рабочими процессами агентов.

`grill-me` развивает идею productivity skills Мэтта Покока:
[`grill-me`](https://github.com/mattpocock/skills/blob/main/skills/productivity/grill-me/SKILL.md)
и [`grilling`](https://github.com/mattpocock/skills/blob/main/skills/productivity/grilling/SKILL.md).
Эта самостоятельная версия не зависит от второго skill: она выводит развилки
из конкретной задачи, спрашивает только о решениях пользователя и направляет
факты, обратимые детали и гипотезы к подходящему способу проверки.

## Для разработки

Каждый skill находится в отдельной папке `skills/<skill-name>/` и может
содержать:

```text
SKILL.md
agents/openai.yaml
references/
scripts/
assets/
```

Для быстрой проверки:

```powershell
python "$env:USERPROFILE\.codex\skills\.system\skill-creator\scripts\quick_validate.py" `
  ".\skills\<skill-name>"
```

Для пробной установки во временную папку:

```powershell
python "$env:USERPROFILE\.codex\skills\.system\skill-installer\scripts\install-skill-from-github.py" `
  --repo dezvin/codex-skills `
  --path skills/<skill-name> `
  --dest "$env:TEMP\codex-skill-test"
```
